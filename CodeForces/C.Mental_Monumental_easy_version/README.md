# [Codeforces] MEX Maximization (Easy Version)

## 📌 문제 개요
- **문제 이름**: MEX Maximization (Easy)
- **핵심 키워드**: MEX, Modulo Operation, Greedy, Binary Search
- **문제 링크**: [Codeforces - MEX Maximization](https://codeforces.com/contest/...) *(링크를 실제 주소로 변경하세요)*

## 📖 문제 설명
정수 배열 $a$가 주어졌을 때, 각 원소 $a_i$에 대해 $a_i = a_i \pmod{b_i}$ 연산을 수행할 수 있습니다. (이때 $b_i$는 임의의 양의 정수). 
연산 후 배열의 **MEX**(배열에 포함되지 않은 최소 비음의 정수)를 최대화하는 문제입니다.

## 💡 핵심 로직 및 제약 조건
1. **나머지 연산의 성질**: 
   - $x \pmod b = y$가 성립하려면, $x=y$이거나 $x \ge 2y+1$이어야 합니다.
   - 즉, 어떤 숫자 $x$를 사용하여 타겟 숫자 $y$를 만들려면 $x$가 충분히 커야 합니다 ($x \ge 2y+1$).

2. **그리디의 함정 (Single-pass의 한계)**:
   - 단순히 작은 타겟($0, 1, \dots$)부터 채우기 위해 눈앞의 숫자를 바로 깎아서 쓰면, 나중에 그 숫자가 원래 값 그대로 필요한 경우(예: $a_i = y$) 실패하게 됩니다.
   - 따라서 "이 숫자를 그대로 쓸 것인가" 아니면 "깎아서 다른 빈 곳을 채울 것인가"를 미리 결정해야 합니다.

3. **이분 탐색 (Binary Search)**:
   - 가능한 MEX의 최대값 $M$을 이분 탐색으로 결정합니다.
   - **Check(M) 함수**:
     - $0$부터 $M-1$까지의 숫자 중 배열에 이미 있는 숫자는 '선점'하여 그대로 둡니다.
     - 남는 숫자들(중복분 또는 $M$ 이상의 큰 수)을 모아 부족한 숫자들을 $x \ge 2y+1$ 조건에 맞춰 채울 수 있는지 확인합니다.
